# Forge AI Build Coach

Version 1.0

## Purpose

The AI Build Coach is the intelligence layer of Forge. The matching engine helps
users find people; the AI Coach helps users make progress. The coach is
responsible for turning ideas into action.

---

## Core Objective

Help users move from: Idea → Validation → Roadmap → Team Formation → Building →
Launch → Growth.

---

## Philosophy

Most AI tools generate information. Forge AI generates momentum. The AI Coach
should always end interactions with a recommended next action. Every
conversation should move a project forward.

---

## Coach Personas

The AI Coach acts as Startup Advisor, Product Manager, Accountability Partner,
Strategic Advisor, and Execution Coach. The AI is not a replacement for users —
it is an accelerator.

---

## Inputs

With permission-based access only, the AI Coach has access to: User Profile,
Project Information, Build Stage, Goals, Skills, Availability, Milestones,
Workspace Data, Team Data, Project History, Files and Documents, User
Preferences.

---

## Primary Capabilities

- **Idea Validation** — Evaluate Problem, Market, Competition, Monetization, Feasibility. Output: Opportunity Summary, Risks, Recommendations.
- **Team Planning** — Determine missing roles, missing skills, recommended collaborators.
- **Roadmap Generation** — 30/60/90-day plans with Milestones, Deliverables, Dependencies.
- **Task Generation** — Weekly tasks, daily actions, launch checklists, prioritized work.
- **Accountability** — Weekly check-ins: What did you accomplish? What blocked progress? What is next?
- **Risk Detection** — Missing skills, inactive members, unclear scope, timeline risk, project stagnation.
- **Launch Planning** — Launch checklist, marketing plan, beta plan, feedback collection plan.

---

## AI Coach Memory

Each project receives a dedicated memory layer. The coach remembers past
decisions, milestones, team discussions, roadmaps, and project evolution.

---

## AI Coach Security

User trust is critical.

**Data Categories:**
- Public Data — Profiles, Public Projects, Public Launches.
- Team Data — Project Workspaces, Messages, Roadmaps, Files.
- Private Data — AI Conversations, Private Notes, Sensitive Files.

**Data Access:** The AI only accesses data authorized by workspace permissions.
Users control access.

**Training Controls:** Users can choose to allow platform learning or "Do not
use my data." The default should prioritize user trust.

---

## AI Interaction Types

- **Builder Chat** — Ask questions, get advice.
- **Project Chat** — AI participates in workspace.
- **Milestone Review** — AI reviews progress.
- **Weekly Review** — AI summarizes accomplishments.
- **Launch Review** — AI evaluates readiness.

---

## Success Criteria

Users should consistently leave conversations with Clarity, Direction,
Confidence, and Next Actions. The coach's job is not conversation — it is
progress.

---

## Future Vision

Long-term, the AI Coach becomes a persistent startup advisor that understands
the user, the team, the project, the goals, and the history better than any
standalone AI tool. The AI becomes the operating system guiding projects from
idea to launch.
