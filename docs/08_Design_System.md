# Forge Design System

Version 1.0

## Design Philosophy

Forge should feel like **Notion + Linear + Product Hunt + Apple** — NOT
Facebook, Instagram, TikTok, Discord, or Reddit.

Forge is a productivity platform, not a social media platform. The emotional
response should be: Professional, Ambitious, Trustworthy, Focused, Motivating.

Builders should feel: *"I'm here to make progress."*

---

## Brand Identity

- **Brand Name:** Forge
- **Tagline:** Build What Won't Build Itself
- **Mission:** Help more people build things that otherwise would never exist.

---

## Visual Principles

1. Clarity over decoration.
2. Progress over engagement.
3. Professional over playful.
4. Action over consumption.

| Style | Size | Weight |
| --- | --- | --- |
| H1 | 40px | Bold |

| Body | 16px | Regular |
| Caption | 14px | Regular |

---

## Border Radius

- Small: 8px
- Medium: 12px
- Large: 16px
- Cards: 20px

## Shadows

Minimal. Subtle elevation only. Avoid heavy shadows. Modern Apple-style depth.

---

## Buttons

- **Primary** — Electric Blue background, white text, rounded corners.
- **Secondary** — White background, blue border, blue text.
- **Destructive** — Red background, white text.

---

## Navigation

**Mobile:** Bottom Tab Navigation — Home, Projects, Matches, Messages, Profile.

**Desktop:** Left Sidebar — Home, Projects, Matches, Messages, Marketplace,
Profile, Settings.

---

## Core Screens

- **Home** — Momentum. AI Next Step, Recommended Builders, Recommended Projects, Active Projects, Milestone Progress.
- **Projects** — Creation. Project Cards, Project Health, Build Stage, Team Members, Milestones.
- **Matches** — Collaboration. Builder Cards, Match %, Why This Match, Connect Button.
- **Messages** — Communication. Inbox style, simple, fast, professional.
- **Profile** — Identity. Bio, Skills, Projects, Interests, Achievements, Build Stage, Builder Archetype.

---

## Card Components

- **Builder Card** — Photo, Name, Role, Skills, Build Stage, Match Score, Connect Button.
- **Project Card** — Project Name, Stage, Health Status, Skills Needed, Team Count, Join Button.
- **Launch Card** — Product, Team, Launch Date, Description, Follow Button.

---

## Project Health UI

Green = Healthy, Yellow = Needs Attention, Red = At Risk. No numerical scores
visible — only status.

## Builder Reputation UI

- **Version 1:** Display *Unproven Builder*. No scores.
- **Version 2:** Reliability, Execution, Collaboration, Expertise — each shown separately. Never combine into a single social score.

## Build Stages UI

Explorer = Gray, Idea = Blue, Validation = Purple, Building = Indigo, Launch =
Green, Growth = Orange, Business = Gold.

---

## Empty States

Extremely important. Every empty state should encourage action:

- "No active projects yet." → Create your first project.
- "No collaborators yet." → Discover builders.
- "No milestones created." → Generate an AI roadmap.

---

## AI Coach Experience

The AI Coach should feel like a trusted advisor, not a chatbot. UI should
emphasize Actions, Plans, Recommendations, Milestones, and Next Steps — not long
conversations.

## Marketplace Design

Focus: celebrating execution. Every launch should feel like a graduation
ceremony. Users should be proud to launch on Forge.

---

## Accessibility

Support Dark Mode, Light Mode, Large Text, Keyboard Navigation, Screen Readers,
WCAG Compliance.

## Design Goal

The platform should constantly communicate one message: **Stop thinking. Start
building.** Everything in the UI should reinforce progress, momentum, and
execution.
