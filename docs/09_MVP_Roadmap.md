# Forge MVP Roadmap

Version 1.0 · Target Timeline: 45 Days

Goal: Launch a production-ready Forge MVP across iOS, Android, and Web.

## MVP Objective

The purpose of the MVP is not to build every feature — it is to validate the
core thesis: *People who want to build things can be matched with the right
collaborators, receive meaningful guidance, and make real progress toward
launching products.*

The MVP succeeds if users can create profiles, discover collaborators, join
projects, create projects, receive AI guidance, form teams, and build momentum.

---

## MVP Success Criteria

Launch Criteria: 100 Users, 20 Projects Created, 10 Teams Formed, 5 Active
Builds, 3 Products Launched, 1 Success Story.

A success story: *"I met people through Forge and built something that would not
have existed otherwise."*

---

## MVP Principles

1. Ship quickly.
2. Prioritize momentum over perfection.
3. Optimize for creation, not engagement.
4. Focus on behavior and outcomes.

---

## Development Phases

### Phase 1 — Foundation (Days 1-5)
Expo app (RN, TS, Expo Router), Supabase project (Auth, DB, Storage, Realtime),
dev/staging/prod environments, GitHub + CI/CD, core DB migration (users,
projects, skills, interests, build stages, archetypes). Completion: users can
register, login, logout, persist sessions.

### Phase 2 — Onboarding (Days 6-10)
Profile setup, skills + levels, interests, archetype, build stage, availability,
goals, partner preferences. Completion: users complete onboarding and create
profiles.

### Phase 3 — Profiles & Discovery (Days 11-15)
Public/editable profile, builder cards, search/filter, home dashboard with
recommendations + AI next step. Completion: users can discover other builders.

### Phase 4 — Projects (Days 16-20)
Project creation/editing/discovery, applications/invitations, project dashboard
(overview, milestones, members, files). Completion: projects can be created and
joined.

### Phase 5 — Matching Engine V1 (Days 21-25)
Builder-to-builder, builder-to-project, project-to-builder matching; weighted
scoring; recommendation explanations. Completion: users receive meaningful
recommendations.

### Phase 6 — AI Build Coach (Days 26-30)
Idea validation, roadmap generation, team recommendations, milestone generation,
risk identification, launch planning; project + personal AI sessions.
Completion: users receive actionable AI guidance.

### Phase 7 — Collaboration (Days 31-35)
Direct messaging, project messaging, workspace chat, file sharing, activity
feed; project workspaces. Completion: teams can collaborate inside Forge.

### Phase 8 — Marketplace (Days 36-40)
Launch marketplace, launch profiles/pages, product discovery, follow launch.
Completion: projects can be published publicly.

### Phase 9 — Polish & Launch (Days 41-45)
Bug fixes, performance, security audit, mobile + web optimization; beta testing
(50-100 users, founder-led onboarding); production release to iOS, Android, Web.

---

## MVP Scope Summary

**Included:** Authentication, Profiles, Onboarding, Projects, Matching,
Messaging, AI Coach, Workspaces, Launch Marketplace, Project Health, Build
Stages, Partner Preferences.

**Excluded:** Advanced Reputation, Investor Marketplace, Recruiting Marketplace,
Machine Learning Matching, Revenue Verification, Advanced Analytics, Enterprise
Features.

---

## MVP North Star

**Active Builds** — 2+ members, active in last 7 days, milestone progress. The
entire MVP should optimize toward increasing Active Builds.
